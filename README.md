# Iris-like Chat Manager Bot

Telegram-бот для управления чатами на заказ — аналог Iris, Зазывала, Анти-спам. Полный цикл модерации, система ролей, статистика активности и развлекательные функции.

## О проекте

Бот разработан на заказ для конкретного заказчика и его сообщества. Полноценный чат-менеджер: модерация, иерархия ролей, статистика, развлекательные команды и защита чата от спама/флуда.

> ⚠️ **О составе репозитория.** По договорённости с заказчиком бот должен работать исключительно в его чатах, поэтому в публичный репозиторий вынесена только часть кодовой базы — модули, демонстрирующие архитектурные решения. Полная версия с привязкой к конкретным чатам и токенам не публикуется по условиям заказа.

## Стек технологий

- **Backend:** Node.js, `node-telegram-bot-api`
- **БД:** PostgreSQL (через `pg`, отдельная схема `iris`)
- **Конфигурация:** `.env` для токенов и подключения к БД

## Реализованный функционал

**Модерация**
- Варны, муты, баны, кики — вручную и автоматически (антиспам, защита от флуда медиа/стикерами/гифками)
- Журнал модерации с историей действий по каждому пользователю

**Роли и права**
- Иерархия: owner → senior_admin → junior_admin → senior_moder → user
- Разграничение доступа к командам в зависимости от роли

**Настройки чата**
- Антиспам, фильтр слов, защита от повторяющихся сообщений
- Возможность временной блокировки чата

**Статистика**
- Подсчёт сообщений, топ активных участников, личная статистика

**Социальные функции**
- Браки, никнеймы, взаимодействия между участниками (обнять, поцеловать и т.д.)
- Мини-игры: монетка, кубик, камень-ножницы-бумага, рулетка

## Технические решения, которые стоит отметить

- **Защита от простоев** — при перезапуске бот игнорирует сообщения, накопившиеся в очереди Telegram во время даунтайма (фильтр по `BOT_START_TIME`), что исключает ложные автобаны при рестарте
- **Единый стиль ответов** — все сообщения бота оформлены через общую функцию `sendHtml()` с `parse_mode: HTML` для консистентного визуального стиля во всех командах
- **Изоляция схемы БД** — таблицы бота живут в отдельной PostgreSQL-схеме (`search_path=iris`), что упрощает деплой на сервер с другими проектами без конфликта имён таблиц

## Структура проекта

```
.
├── index.js     ← основная логика команд и обработчиков
├── db.js        ← подключение к PostgreSQL и запросы
└── .env.example ← пример переменных окружения
```

## Деплой

Бот запущен в проде на VPS через PM2:

```bash
pm2 start index.js --name iris-bot
pm2 save
pm2 startup
```
